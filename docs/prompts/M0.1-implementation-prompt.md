# M0.1 Implementation Prompt

Use only after the M0.1 plan has passed the independent plan judge and the product owner has explicitly authorized implementation.

Implement **M0.1 — Repository Foundation** exactly as frozen and planned.

## Rules

- Read the frozen spec, repository audit, approved plan, and judge report first.
- Reproduce the baseline before editing.
- Preserve unrelated changes.
- Implement only the authorized checkpoint if the plan is split into checkpoints.
- Do not add media ingest, FFmpeg execution, file picker, transcript UI, AI calls, persistence, timeline preview, or exports.
- Keep the renderer sandboxed and free from Node integration.
- Expose only narrow typed preload methods.
- No generic shell or unrestricted filesystem API.
- Write tests for every new domain invariant and architecture guard.
- Update `M0.1-implementation-report.md` with actual evidence, not planned evidence.
- Stop after the authorized checkpoint and verification.

## Final response

Return:

- branch / HEAD;
- worktree before and after;
- exact files changed;
- implementation summary;
- dependency rationale;
- test/typecheck/lint/build results;
- security assertions;
- deviations or limitations;
- next gate, without starting it.
