## Delivery strategy

M1.0 is implemented in seven reviewable checkpoints. Each checkpoint:

1. starts from a clean tracked worktree;
2. receives separate product-owner authorization;
3. writes failing tests or executable guards before implementation where technically possible;
4. changes only its declared ownership boundary;
5. runs frozen install plus checkpoint-targeted and complete repository checks;
6. updates `M1.0-implementation-report.md` with exact commands, results, files, deviations, retries, and worktree state;
7. creates one local checkpoint commit;
8. stops before the next checkpoint;
9. does not push unless publication is separately authorized.

Rollback means reverting the complete local checkpoint commit, including its documentary evidence. No destructive reset or force operation is planned.

## Target architecture

```text
Renderer
  -> frozen semantic preload methods (no paths)
  -> strict main-process IPC schemas
  -> in-memory ingest controller
       -> native dialogs + bounded transcript read
       -> Electron utility-process client
            -> versioned media-probe worker message
            -> fixed-argument ffprobe spawn (shell: false)
  -> renderer-safe snapshot polling (>= 250 ms)
```

Absolute media/transcript paths, executable resolution, environment, raw `ffprobe` output, stderr, process IDs, diagnostics, and utility-process handles remain on the privileged side.

The lockfile changes only for new workspace edges unless compatibility work demonstrates an unavoidable requirement. Any registry dependency requires a plan amendment and explicit approval before installation.

## Repository guard evolution

The current guard encodes M0.1 names and prohibitions. The M1.0 guard must evolve test-first while preserving all security boundaries:

- add `transcript` to framework-independent packages;
- allow `transcript -> domain` and runtime `media -> domain` edges;
- continue forbidding Node built-ins, Electron, React, app sources, provider SDKs, FFmpeg wrappers, database/packaging dependencies, undeclared edges, and cross-package relative imports in pure packages;
- continue requiring `export` to expose no symbol;
- continue forbidding `packages/fixtures`, binary content, binary extensions, Git LFS pointers/configuration, and CI media downloads;
- permit Node/process/filesystem imports only under specifically owned Electron main/worker source, not preload/renderer or pure packages;
- add negative cases proving a similarly named directory cannot bypass the privileged-source allowlist;
- replace misleading “forbidden in M0.1” diagnostics with current policy language.

Guard weakening is a blocking failure.

## Closed result-code traceability

The following matrix is normative for this corrected plan when Amendment 001 is approved. “Positive” means the condition deterministically produces the listed code/outcome. “Negative” means an adjacent condition cannot be misclassified as that code; terminal-latch race tests are additionally required wherever events can coincide.

| Result or outcome | Owner | CP | Required positive test | Required negative / precedence test |
|---|---|---:|---|---|
| `INVALID_REQUEST` | shared renderer/main IPC schemas | 5 | wrong version, extra key, or wrong type rejects before work | valid closed request proceeds; no dialog, file, worker, or spawn occurs after rejection |
| `DIALOG_CANCELLED` | main dialog adapter | 4 | closed dialog and empty selection return cancellation | selected path proceeds; cancellation wins before later file work |
| `FILE_UNAVAILABLE` | main file adapter | 4 | canonicalize/stat/open failure maps here | regular and non-regular targets are distinguished; no later code replaces it |
| `FILE_NOT_REGULAR` | main file adapter | 4 | directory/device-like stat maps here | inaccessible path remains `FILE_UNAVAILABLE`; regular file proceeds |
| `MEDIA_EMPTY` | main media file adapter | 4 | zero-byte regular media maps here | zero-byte transcript is parsed normally; non-empty media proceeds |
| `MEDIA_UNSUPPORTED` | worker probe runner | 3, 5 | normal probe nonzero numeric exit maps here | spawn error/signal remains `PROBE_FAILED`; stderr text is ignored for classification |
| `FFPROBE_CONFIGURATION_INVALID` | main executable configuration | 4 | relative, empty, oversized, or NUL override rejects before worker creation | absent override yields bare `ffprobe`; valid absolute string is passed unchanged |
| `FFPROBE_NOT_FOUND` | worker version/probe runner | 3, 5 | spawn `ENOENT` for version or probe maps here | other spawn errors map to incompatible/probe failure; raw executable stays private |
| `FFPROBE_INCOMPATIBLE` | worker version runner | 3, 5 | timeout, cap, nonzero, signal/OS error, invalid UTF-8, or invalid first line maps here | exact compatible line passes; cancellation outcome wins its race |
| `PROBE_TIMEOUT` | worker probe runner | 3 | 30-second timer latches before kill | completed/cancelled/capped probe cannot later become timeout |
| `PROBE_CANCELLED` | worker probe runner | 3, 5 | cancellation latches before abort/kill | earlier typed terminal result wins; late close/error is ignored |
| `PROBE_OUTPUT_LIMIT` | worker probe runner | 3 | either pipe crossing its cap latches before kill | at-cap output proceeds; timeout/cancel already latched is retained |
| `PROBE_FAILED` | worker probe runner | 3, 5 | non-`ENOENT` spawn error or unrequested signal maps here | numeric nonzero exit remains `MEDIA_UNSUPPORTED` |
| `PROBE_OUTPUT_INVALID` | worker decode plus pure media parser | 2, 3, 5 | invalid UTF-8/JSON/required structure or required field type maps here | optional invalid rational remains unavailable with warning, not failure |
| `VIDEO_STREAM_MISSING` | pure media parser | 2 | no eligible non-attached primary video maps here | attached picture is excluded; eligible video proceeds |
| `AUDIO_STREAM_MISSING` | pure media parser | 2 | no primary audio maps here after video succeeds | missing video has earlier precedence; valid audio proceeds |
| `MEDIA_DURATION_INVALID` | pure media parser | 2 | no valid format or selected-stream duration maps here | valid format wins; greatest valid selected-stream fallback succeeds with warning |
| `TRANSCRIPT_PREREQUISITE_MISSING` | main ingest controller | 4, 5 | absent/mismatched active ready asset rejects before dialog | exact active asset ID proceeds; stale well-formed results are ignored |
| `TRANSCRIPT_TOO_LARGE` | main transcript reader | 4 | stat size above 20 MiB rejects before read | exact limit is read; filesystem failure retains earlier file code |
| `TRANSCRIPT_ENCODING_INVALID` | main decoder / transcript boundary | 1, 4 | fatal UTF-8 failure or replacement character maps here | valid UTF-8 continues to JSON; cancellation already latched wins |
| `TRANSCRIPT_JSON_INVALID` | pure transcript parser | 1 | syntax/root/trailing-data failure maps here | valid object with bad schema reaches schema code |
| `TRANSCRIPT_SCHEMA_UNSUPPORTED` | pure transcript parser | 1 | absent, non-integer, or non-`1` schema maps here | literal `1` proceeds to limits/shape |
| `TRANSCRIPT_LIMIT_EXCEEDED` | pure transcript parser | 1 | entry/text/string/object limit breach maps here | values at each limit proceed; schema failure retains precedence |
| `TRANSCRIPT_ENTRY_INVALID` | pure transcript parser | 1 | remaining shape/type/confidence/text/time/interval failure maps here | valid entries proceed to adjacent scan; limit breach remains earlier |
| `TRANSCRIPT_ORDER_INVALID` | pure transcript parser | 1 | adjacent start decreases maps here | same pair that also overlaps reports order, not overlap |
| `TRANSCRIPT_OVERLAP_UNSUPPORTED` | pure transcript parser | 1 | ordered adjacent start before previous end maps here | equal boundary and gaps pass; order failure remains earlier |
| `TRANSCRIPT_OUT_OF_BOUNDS` | pure transcript parser | 1 | valid converted end exceeds active duration | end equal to duration passes; earlier pair failure retains precedence |
| `TRANSCRIPT_CANCELLED` | main transcript reader/controller | 4, 5 | cancellation latches before read/decode/parse completion | earlier terminal result wins; late read result is ignored |
| `WORKER_UNAVAILABLE` | main utility client | 4, 5 | fork/channel failure or premature exit/disconnect maps here | typed terminal result already latched is retained |
| `INTERNAL_ERROR` | privileged protocol/invariant boundary | 3–5 | injected unexpected invariant/uncategorized exception maps to fixed safe failure | every listed expected condition maps to its specific code; diagnostics never cross IPC |
