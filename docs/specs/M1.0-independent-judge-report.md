## Blocking findings

| Severity | Finding | Evidence and frozen-contract conflict |
|---|---|---|
| Blocking / high | Bounded process completion resolves before subprocess close after cancellation, timeout, or output-limit termination. | `apps/desktop/src/worker/media-probe/bounded-process.ts` removes the `close` listener, resolves the promise, and only then calls `kill()`. An independent fake-child reproduction returned `cancelled` with one kill while `closeEmitted` remained false. The frozen cleanup contract requires termination/close to complete safely; the current ordering cannot prove that no child remains and allows overlap with a later job. Existing tests encode early resolution rather than close-wait behavior. |
| Blocking / high | Media cancellation is not latched across utility-client configuration and controller replacement. | `apps/desktop/src/main/media-probe-client.ts` posts `cancel` but neither marks nor settles the pending job. `apps/desktop/src/main/ingest-controller.ts` immediately clears its active operation and restores stable state. Independent reproduction of cancel-before-configuration produced `configure → cancel(job-1) → probe(job-1)`; a subsequent replacement returned `INTERNAL_ERROR` because the earlier client request remained pending. This violates cancellation-before-work, stale-job suppression, replace/retry acceptance, and the rule that expected conditions cannot become `INTERNAL_ERROR`. |
| Blocking / medium | Renderer ready-state metadata and warnings are incomplete. | `apps/desktop/src/renderer/App.ts` displays duration, size, dimensions/video codec, average frame rate, audio codec, and container, but omits real/base frame rate, time base, rotation, and all closed warning codes. The frozen M1.0 outcome requires rational rate metadata and relevant warnings in the renderer summary. |

The two cancellation/process findings also prevent the separately gated manual smoke from proving that cancellation leaves no child `ffprobe` process.

## Nonblocking findings

1. The shared renderer error schema admits `DIALOG_CANCELLED`, `PROBE_CANCELLED`, and `TRANSCRIPT_CANCELLED`, although the normative contract classifies them only as cancellation outcomes. Current product paths do not emit them as errors, but the closed boundary is broader than specified.
2. Renderer retry routing tests only `error.code.startsWith('TRANSCRIPT_')`. Transcript-selection `FILE_UNAVAILABLE` or `FILE_NOT_REGULAR` therefore offers “Retry video” despite preserved media state.
3. At the reviewed candidate, README and implementation-report gate text still described CP7 publication and independent judgment as pending. This documentary evidence update records the actual state; it does not correct executable findings.
4. A user-owned untracked directory appeared during the audit. The judge did not inspect or use it. Tracked state stayed unchanged and synchronized, but the overall worktree is not clean enough for freeze.
