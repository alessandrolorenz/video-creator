# M1.0 — Transcript Selected Cut Vertical Slice

Status: Early draft — not ready for implementation

## Objective

Prove the product’s core value with one local video and one timed transcript: selected text becomes a validated, previewable, rendered cut.

## User story

As a creator, I provide a source video, its timed transcript, and the passages I want to keep. The application finds each passage, shows ambiguities, constructs a sequence in my chosen order, lets me adjust boundaries, and exports the result.

## Proposed checkpoints

### M1.0A — Media metadata and transcript import

- one source asset;
- ffprobe metadata;
- word- or segment-timed transcript JSON;
- normalized in-memory project;
- no rendering.

### M1.0B — Text selection resolution

- selected passages input;
- exact normalized matching first;
- ordered fuzzy fallback behind explicit confidence;
- repeated phrase ambiguity;
- no silent best guess under ambiguity threshold;
- source ranges include configurable pre/post handles.

### M1.0C — Simple timeline and preview

- hard-cut sequence;
- reorder;
- trim in/out;
- slip within handles;
- source-context preview;
- deterministic invariants.

### M1.0D — Render and export

- cancelable local render;
- progress and errors;
- MP4 output;
- internal timeline JSON;
- FCP7 XML simple-cut export;
- real import smoke in Premiere and Resolve.

## Matching rules to freeze later

- Unicode normalization and diacritics policy;
- punctuation policy;
- whitespace policy;
- case policy;
- exact-vs-fuzzy thresholds;
- repeated phrase disambiguation;
- selection ordering;
- overlap merge policy;
- sentence-boundary handles;
- transcript/source alignment confidence.

## Failure states

- unsupported media;
- missing video or audio stream;
- invalid transcript timestamps;
- transcript exceeds asset duration;
- selection not found;
- ambiguous selection;
- invalid range after trim/slip;
- render canceled;
- render failed;
- output path unavailable;
- exporter warning or unsupported timeline state.

## MVP quality bar

The application must never silently produce a cut from an unresolved ambiguous passage. It must show the user the candidate occurrences and require a deterministic choice.
