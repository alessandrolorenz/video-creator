## Authority and scope correction

This proposal replaces the unfrozen early draft that incorrectly combined M1.0 through M1.3 into one vertical slice. It aligns M1.0 with the M0.0-frozen roadmap:

- M1.0 imports and validates one media asset and one timed transcript;
- M1.1 resolves selected text and ambiguity;
- M1.2 builds and previews a timeline;
- M1.3 renders and exports.

M1.0 produces no cut. The broader transcript-selected-cut MVP remains the outcome of completing all four milestones.

## Objective

Prove the first privileged product workflow without weakening the M0.1 security boundary: a user selects one local MP4/MOV-family video and one versioned timed-transcript JSON file; the application probes, validates, binds, and displays a sanitized in-memory summary.

## User outcome

As a creator, I can select one local interview/talking-head video and its timed transcript and know whether they form a valid source pair before selecting passages or editing anything.

The ready state answers:

- which media display name was selected;
- whether required video and audio streams exist;
- duration, dimensions, codec names, and rational rate metadata;
- transcript granularity, language when supplied, entry count, and covered time range;
- whether every transcript interval is valid and inside the asset duration;
- warnings relevant to later preview/render work, without pretending those later capabilities exist.

## In scope

- one active local media asset selected through a native main-process dialog;

## Canonical time rule

All canonical times are `TimeUs` safe integers from `packages/domain`.

Accepted input units and factors are:

| Input unit | Conversion to microseconds |
|---|---:|
| `microseconds` | `round(value)` |
| `milliseconds` | `round(value × 1_000)` |
| `seconds` | `round(value × 1_000_000)` |

Input values must be finite, non-negative numbers. The converted value must be a JavaScript safe integer. Rounding is the normal JavaScript `Math.round` rule and occurs only once at the import boundary. Canonical values are never stored as floating-point seconds.

An interval is valid only when `endUs > startUs` after conversion. M1.0 never clamps, sorts, merges, expands, or otherwise repairs timing silently.

## Timed transcript input contract

The only accepted input is UTF-8 JSON with this logical shape:

```ts
interface TimedTranscriptInputV1 {
  readonly schemaVersion: 1;
  readonly granularity: 'word' | 'segment';
  readonly timeUnit: 'microseconds' | 'milliseconds' | 'seconds';
  readonly language?: string;
  readonly entries: readonly TimedTranscriptEntryInputV1[];
}

interface TimedTranscriptEntryInputV1 {
  readonly text: string;
  readonly start: number;
  readonly end: number;
  readonly speakerId?: string;
  readonly confidence?: number;
}
```

No extra object keys are accepted in V1.

### Input limits

- raw file: at least 2 bytes and at most 20 MiB;
- entries: 1 through 250,000;
- entry text: non-empty after trimming, at most 16,384 UTF-16 code units;
- optional language: 1 through 64 trimmed code units;
- optional speaker ID: 1 through 256 trimmed code units;
- optional confidence: finite number from 0 through 1 inclusive;
- total decoded entry text: at most 20,000,000 UTF-16 code units.

The raw file must decode as valid UTF-8 without replacement characters and parse as one JSON object. JSON arrays, primitives, duplicate top-level documents, non-finite numeric tokens, and trailing non-whitespace data are invalid.

### Ordering and overlap

The input array order is authoritative. Converted intervals must be monotonically non-overlapping:

- each entry starts at or after the previous entry start;
- each entry starts at or after the previous entry end;
- equal boundaries are allowed (`previous.endUs === next.startUs`);
- overlapping speech/transcript lanes are not supported by V1 and are rejected;
- ties that would require silent reordering are rejected.

### Canonical transcript document

The parser produces a versioned immutable `TranscriptDocumentV1` with:

- opaque document ID generated by the privileged application;
- active opaque asset ID supplied by privileged state, never trusted from the file;
- preserved granularity and optional language;
- canonical entries with deterministic IDs `entry-000001`, `entry-000002`, and so on;
- original text preserved byte-for-decoded-string value; no matching normalization yet;
- `TimeUs` start/end values;
- optional speaker ID and confidence;
- covered range from first start through last end;
- entry count.

Every entry must end at or before the active asset `durationUs`. Exceeding the duration is a hard validation error; M1.0 does not clamp it.

## Media asset contract

Privileged state owns a `MediaSourceRefV1` containing an opaque asset ID and canonical absolute path. This object never crosses to preload or renderer.

The renderer-safe immutable `MediaAssetSummaryV1` contains:

- schema version `1`;
- opaque asset ID;
- display name only, with no directory information;
- byte size;
- canonical duration in `TimeUs`;
- container format names returned by `ffprobe`, sanitized as data;
- primary video: stream index, codec name, coded width/height, display rotation when available, average frame-rate rational, real/base frame-rate rational, and stream time-base rational;
- primary audio: stream index, codec name, sample-rate integer when available, and channel count when available;
- zero or more closed warning codes.

The summary contains no absolute path, parent directory, environment value, executable location, raw `ffprobe` payload, command, arguments, stderr, or process identifier.

### Deterministic stream and duration rules

- Streams are ordered by numeric `index` ascending after strict validation.
- Primary video is the first video stream that is not marked as an attached picture.
- Primary audio is the first audio stream.
- At least one primary video and one primary audio stream are required for M1.0.
- Duration uses valid positive container `format.duration` when present; otherwise it uses the greatest valid positive duration among the primary video and audio streams.
- Decimal-second durations are converted once with the canonical seconds rule.
- Missing or invalid duration after fallback is a hard error.
- `avg_frame_rate`, `r_frame_rate`, and `time_base` are preserved as reduced positive integer rationals when valid; missing/invalid values produce explicit warnings or errors according to whether later timing would be impossible.
- Variable-frame-rate suspicion is a warning, not a claim of support or a hard ingest failure. M1.0 does not calculate frame-accurate edit boundaries.

V1 warning codes are closed to:

- `DURATION_FALLBACK_USED`;
- `VARIABLE_FRAME_RATE_SUSPECTED`;
- `RATE_METADATA_MISSING`;
- `ROTATION_METADATA_PRESENT`.

## External `ffprobe` prerequisite

M1.0 uses an external local `ffprobe` executable for development and manual smoke.

- It is not a pnpm production dependency.
- It is not downloaded, bundled, copied, packaged, signed, or redistributed.
- Distribution/license decisions remain M4.
- The audited host currently has no `ffprobe` on `PATH`; this must be satisfied before real manual smoke, with separate authorization for any machine installation.
- Resolution is privileged and deterministic: optional absolute main-process-only development configuration, otherwise the inherited executable name `ffprobe`.
- The renderer cannot provide, inspect, or change the executable or environment.
- First use validates `ffprobe -version`; the app records only a sanitized version line and capability state.

Missing or incompatible `ffprobe` yields an actionable prerequisite error and never triggers automatic installation.

## Probe-worker boundary

1. Electron main owns native file dialogs, canonical paths, and job lifecycle.
2. A dedicated Electron utility process owns media probing; it accepts only a strictly parsed versioned probe-job message.
3. The utility process invokes `ffprobe` asynchronously with a fixed argument template equivalent to `-v error -show_format -show_streams -of json <absolute-selected-path>`.
4. Invocation uses an argument array, `shell: false`, ignored stdin, bounded stdout/stderr, and no renderer-controlled argument.
5. Default timeout is 30 seconds.
6. Stdout is capped at 8 MiB and stderr at 1 MiB. Exceeding either cap terminates the process and fails the job.
7. Cancellation uses the job ID and aborts the subprocess; success cannot be emitted after cancellation wins.
8. Completion waits for process close and validates exit code, signal, output decoding, JSON structure, semantic metadata, and job identity exactly once.
9. Utility-process crash/exit rejects the active job and is observable as a typed error; it never leaves a pending renderer promise.
10. Window close or app shutdown cancels active jobs and terminates the utility process.

The worker contract is media-specific. It is not a generic command runner.

## Native file import boundary

- Media and transcript dialogs run only in Electron main, attached to the current window.
- The renderer invokes semantic methods such as `chooseMediaAsset`, `cancelMediaImport`, `chooseTimedTranscript`, and `cancelTranscriptImport`; it never supplies a path.
- Dialog cancellation returns a typed cancelled result, not an error.
- Selected paths are canonicalized and verified as readable, non-empty regular files before work.
- Media filters advertise `mp4`, `mov`, and `m4v`; transcript filters advertise `json`. Filters are UX aids, not trust boundaries.
- Replacing media invalidates any previously imported transcript atomically.
- Importing a transcript requires one ready active asset.
- Transcript file reading is bounded before allocation/parsing and is cancellable.

## IPC and preload contract

All renderer requests contain only `contractVersion: 1` plus an opaque job/asset ID when required. Every request enters main as `unknown` and passes a strict runtime schema before dialog, file, worker, or process activity.

The preload exposes a single frozen namespace. In addition to the existing foundation status, its M1.0 surface is exactly:

- `chooseMediaAsset()` — opens the native dialog and resolves to a typed `cancelled` or `started` result containing only the opaque job ID;
- `cancelMediaImport(jobId)` — requests cancellation of that exact active media job;
- `chooseTimedTranscript(assetId)` — opens the native dialog for the exact active asset and resolves to `cancelled` or `started` with an opaque job ID;
- `cancelTranscriptImport(jobId)` — requests cancellation of that exact active transcript job;
- `getIngestSnapshot()` — returns one strictly validated renderer-safe snapshot.

While a job is active, the renderer polls `getIngestSnapshot()` at a bounded interval no shorter than 250 milliseconds and stops polling in a stable state. M1.0 exposes no event emitter, channel subscription, or generic callback registration.

No raw `ipcRenderer`, event emitter, channel string, callback registration primitive, filesystem handle, full path, process handle, command, argument array, environment object, or secret is exposed.

Responses are closed discriminated unions. Error messages are user-safe and path-redacted; diagnostic causes remain privileged.

## In-memory ingest state

The privileged state machine has these states:

- `empty`;
- `choosing-media`;
- `probing-media`;
- `media-ready`;
- `choosing-transcript`;
- `validating-transcript`;
- `ready`;
- `cancelled` as an operation result returning to the last stable state;
- `error` with the last stable valid state preserved when safe.

Only one media job and one transcript job may be active. Late results from replaced/cancelled job IDs are ignored. State is in memory only and resets on application restart.

## Test requirements

### Domain and transcript units

- all unit conversions, half-step rounding boundaries, overflow, NaN/infinity, negative values, and zero-length-after-rounding;
- strict schema version/granularity/unit and extra-key rejection;
- UTF-8, JSON, raw-size, entry-count, total-text, field-length, and confidence limits;
- empty/whitespace text, optional fields, deterministic entry IDs, and preservation of source text;
- monotonic order, equal boundaries, gaps, overlap, reversed ranges, and asset-duration bounds;
- serialization/immutability where serialization is introduced.

### Media parser and worker units

- valid minimal captured `ffprobe` JSON for MP4 and MOV represented as text, not binary media;
- stream ordering, attached-picture exclusion, required streams, duration fallback, rational parsing/reduction, rotation, and VFR warnings;
- spawn executable/arguments are exact; shell is disabled; stdin ignored;
- missing executable, error/close races, nonzero exit, signal exit, malformed/oversized output, timeout, cancellation, late events, and utility-process exit;
- output and renderer errors redact paths, args, raw stderr, and diagnostics.

### Electron and IPC units/integration

- every request schema accepts only its closed shape and rejects invalid input before work;
- native dialogs are main-owned, single-selection, parented, and filtered;
- paths never cross preload/renderer contracts;
- bridge is frozen and exposes only the approved semantic methods;
- cancel/replace/job-ID races preserve the state machine;
- window/app close cancels active work;
- renderer covers every stable, progress, cancelled, ready, and error state;
- security assertions continue proving sandbox, context isolation, no Node globals, local content, restrictive CSP, popup denial, and navigation guards.

### Architecture and repository guards

- `packages/transcript` may depend only on `packages/domain`;
- framework-independent packages still reject Electron, React, Node built-ins, app sources, provider SDKs, FFmpeg wrappers, and undeclared edges;
- only the specifically owned privileged adapter/worker may import the minimum process/filesystem APIs;
- no generic command or filesystem bridge appears;
- no binary fixture, `packages/fixtures`, Git LFS, CI download, packaging dependency, AI SDK, database, timeline/export implementation, or unexpected generated file appears.

### Fixture strategy

Normal tests use TypeScript/JSON text literals, temporary text files, and fake process/worker adapters. No committed media is needed. Manual smoke inputs stay outside the repository. The existing zero-byte binary-fixture budget remains unchanged.

## Acceptance criteria

M1.0 passes only when:

1. the corrected scope contains ingest/validation only and no M1.1–M1.3 behavior;
2. one real local media asset can be selected, securely probed, and represented by validated canonical metadata;

4. renderer-visible state contains no path or privileged capability;
5. probe/transcript jobs are bounded, cancellable, race-safe, and cleaned up on replacement/window/app close;
6. every closed error and UX state is tested;
7. architecture/fixture guards evolve without weakening pure-package, security, binary, AI, database, packaging, timeline, or export boundaries;
8. clean-room install and the complete automated sequence pass;
9. hosted CI passes for the reviewed commit;
10. an independent implementation judge passes scope, contracts, security, worker/process, fixtures, tests, CI, and worktree state;
