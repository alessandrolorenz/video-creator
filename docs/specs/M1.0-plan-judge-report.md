## Blocking findings

1. **Media/worker result taxonomy is not deterministic.** There is no closed condition/precedence matrix for `MEDIA_UNSUPPORTED` versus `PROBE_FAILED`, `FFPROBE_INCOMPATIBLE`, `WORKER_UNAVAILABLE`, `INTERNAL_ERROR`, or coincident transcript order/overlap failures.
2. **Rational and warning algorithms are incomplete.** Required fields, error-versus-`RATE_METADATA_MISSING`, exact `VARIABLE_FRAME_RATE_SUSPECTED`, and `ROTATION_METADATA_PRESENT` triggers are not frozen.
3. **The `ffprobe -version` flow is not test-first.** Exact args, executable-not-found, nonzero exit, invalid output, incompatible version, sanitized line, and stdout/stderr boundary tests are missing.
4. **Executable-configuration ownership conflicts.** CP3 cannot resolve a configuration that the spec defines as main-only when CP3 owns no main-process file.
5. **CP4 is not small enough.** It crosses utility client, dialogs/read, state controller, IPC, preload, and shutdown boundaries.

## Non-blocking notes

- The M0.1 security foundation remains suitable: sandbox, context isolation, narrow preload, CSP, and local navigation controls are implemented and tested.
- Electron `utilityProcess` is viable with the locked Electron/build baseline if the worker target is explicitly included in build and typecheck.
- The current guard still emits M0.1-specific messages, and the plan already requires their correction.
- CP1's pure logic is technically coherent in isolation, but the frozen workflow prohibits starting it before the entire plan passes Gate 4.

## Scope-creep findings

None. The plan does not add matching, timeline, preview, render/export, AI, persistence, packaging, multiple assets, registry dependencies, or binary fixtures.

## Required corrections

1. Approve a specification amendment that defines a complete condition → result/error matrix and precedence.
2. Freeze exact rational, warning, VFR, rotation, and `ffprobe` version compatibility algorithms.
3. Add requirement/code → owner → checkpoint → positive/negative test traceability.
4. Move absolute executable-configuration resolution to a main-owned checkpoint; the worker receives only already-resolved privileged configuration through a strict contract.
5. Require complete version-check tests and a compiled-worker/real-utility-process integration using a fake executable and no media binary or real `ffprobe`.
6. Split CP4 at minimum into privileged adapters/client/controller and IPC/preload/lifecycle checkpoints.
