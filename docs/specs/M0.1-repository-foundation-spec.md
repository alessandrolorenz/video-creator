# M0.1 — Repository Foundation Specification

Status: Approved / Frozen — amended by M0.1 Amendment 001 on 2026-07-21

## Objective

Create a minimal, secure, testable desktop repository foundation that can support later media editing without implementing product functionality.

## In scope

- pnpm TypeScript monorepo;
- Electron main/preload/renderer split;
- React renderer with a single neutral shell screen;
- strict TypeScript settings;
- typed IPC request/result/error envelope;
- worker-job interface with cancellation contract;
- domain package containing branded `TimeUs`, `SourceRange`, and validators;
- AI-contract package containing provider-neutral placeholder interfaces only;
- media package containing interfaces only—no FFmpeg execution;
- timeline package containing an empty versioned timeline contract and validators;
- export workspace boundary containing package metadata and an empty barrel only, with no public export contracts or implementation;
- repository fixture policy at `docs/fixture-policy.md` defining approved locations, provenance/license records, size budgets, versioning/checksums, privacy restrictions, synthetic-data preference, binary-media review requirements, and milestone ownership; M0.1 adds no binary media fixture;
- Vitest or equivalent test baseline;
- lint/format/typecheck/build scripts;
- CI workflow;
- architecture dependency guards where practical;
- documentation links in root README.

## Explicit non-goals

- importing media;
- file picker;
- transcript UI;
- FFmpeg or ffprobe execution;
- API calls;
- persistence/database;
- timeline preview;
- export request/result contracts, serializers, interchange formats, file I/O, or generated exports;
- product styling;
- authentication;
- auto-update;
- installer packaging.

## Security requirements

- renderer has no Node integration;
- context isolation enabled;
- sandbox enabled where supported;
- no remote content;
- preload exposes only explicitly typed methods;
- IPC handler validates inputs at runtime;
- no API key is accessible to renderer code;
- no generic `execute`, `shell`, `readFile`, or unrestricted path API.

## Domain requirements

`TimeUs` and `SourceRange` validators must prove:

- non-negative start;
- end greater than start;
- duration helper;
- containment helper;
- no floating-point equality assumptions in tests.

## Verification

- clean install;
- lint pass;
- application typecheck pass;
- test pass;
- renderer build pass;
- Electron main/preload build pass;
- security configuration test or static assertion;
- dependency-boundary test;
- no production media or AI dependency added.
- `docs/fixture-policy.md` exists, is linked from the root README, contains every section required by Amendment 001, and no binary media fixture or Git LFS configuration is introduced.
